# journal-portal-space

本仓库用于归档与发布多个独立的 HTML 页面。这些页面各自独立运行，不依赖任何特定后端服务，也不针对单一网站或域名。

## 项目简介

`journal-portal-space` 是一个静态页面集合仓库，主要存放各类独立编写的 HTML 文件。每个页面可以视为一个自包含的“入口”或“记录”，适合用于展示、存档或临时发布。仓库本身不提供搜索、索引或动态功能，仅作为文件托管与版本管理的中转空间。

## 目录说明

```
journal-portal-space/
├── README.md          # 本说明文件
├── index.html         # 可选的入口页面（若存在）
└── pages/             # 存放独立 HTML 页面的目录（建议）
    ├── example.html
    └── ...
```

- `index.html`：若仓库根目录下存在此文件，通常作为默认访问页面。
- `pages/`：建议将各个独立的 HTML 页面放置于此目录下，以便于管理。
- 其他文件或目录可根据需要自行组织，保持整洁即可。

## 页面归档说明

仓库中的每个 HTML 页面均为独立的归档单元。归档时请注意：

- 所有资源（CSS、JavaScript、图片等）建议内联或通过相对路径引用，避免依赖外部链接。
- 页面标题、描述等元信息应写在 `<head>` 中，便于识别。
- 文件名建议使用有意义的英文或拼音，避免特殊字符。

## 维护说明

1. **添加新页面**：将 HTML 文件放入 `pages/` 目录（或根目录），提交并推送。
2. **更新已有页面**：直接修改对应文件，提交新版本。建议在 commit message 中简要说明变更内容。
3. **删除页面**：移除文件并提交。若页面已被外部引用，请提前通知相关方。
4. **版本控制**：所有历史记录均通过 Git 保留，可随时回退至旧版本。
